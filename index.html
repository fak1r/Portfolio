Калькулятор

Стартовый проект на JS

Все буквы автоматически заменяются, как при вводе с клавиатуры, так и при попытке вставить извне. Кнопка блокируется до ввода новых значений или смены селектора. Калькулятор реализован через объект calc, в котором хранятся 4 арифметические функции. При клике на кнопку вызывается функция calculate(), которая ищет арифметическое действие в объекте calc по ключу и передаёт в найденную функцию 2 числа из инпутов.

Что такое замыкание?

Замыкание – это функция, которая запоминает свои внешние переменные и может получить к ним доступ. Все функции в JavaScript являются замыканиями и «при рождении» получают скрытое свойство [[Environment]], которое ссылается на лексическое окружение места, где они были созданы. Другими словами, функция навсегда запоминает ссылку на лексическое окружение, где она была создана. И [[Environment]] – скрытое свойство функции, которое содержит эту ссылку.

Какие бывают типы данных?

В JavaScript есть 8 основных типов данных:
number для любых чисел: целочисленных или чисел с плавающей точкой; целочисленные значения ограничены диапазоном ±(253-1).
bigint для целых чисел произвольной длины.
string для строк. Строка может содержать ноль или больше символов, нет отдельного символьного типа.
undefined для неприсвоенных значений – отдельный тип, имеющий одно значение undefined.
null для неизвестных значений – отдельный тип, имеющий одно значение null.
boolean для true/false.
Symbol для уникальных идентификаторов.
object для более сложных структур данных.

Что такое promise?

Это специальный объект в JavaScript, который связывает «создающий» и «потребляющий» коды вместе. Когда промис получает результат, он переходит в состояние resolve(value) — если работа завершилась успешно, с результатом value или reject(error) — если произошла ошибка, error – объект ошибки.
Внутренние свойства промиса: state («состояние») — вначале "pending" («ожидание»), потом меняется на "fulfilled" («выполнено успешно») при вызове resolve или на "rejected" («выполнено с ошибкой») при вызове reject. result («результат») — вначале undefined, далее изменяется на value при вызове resolve(value) или на error при вызове reject(error).

Что такое Event Loop?

Event loop - событийный цикл - бесконечный цикл, в котором движок JavaScript ожидает задачи, исполняет их и снова ожидает появления новых. Пока есть задачи: 1. Выполнить их, начиная с самой старой. 2. Бездействовать до появления новой задачи, а затем перейти к пункту 1

Базовый таймер в секундах
class Timer - таймер обратного отсчёта реализован с применением ООП. Выводит время в секундах.
Имеет методы: start() stop() tick() render()
Первый наследник
class FormatTimer extends Timer - преобразует секунды в правильный формат за счёт новой функции timeFormat(), которой не было у родительского класса. Так же переопределяет родительский метод render(), для корректного отображения времени по формату.
Наследник третьего поколения
class WordTimer extends FormatTimer - добавляет функцию wordsTime(), которая производит буквенное склонение едениц измерения времени.
Управление таймерами
Кнопки управления завязаны на все таймеры одновременно. Чтобы таймеры появились, нужно установить время в секундах.
Добавлена валидация инпута, вводить можно только целые числа.
